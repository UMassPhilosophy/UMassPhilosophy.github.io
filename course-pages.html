Creating and updating course listings

We should maintain a total of four separate files per semester. Two of them are for scheduling information (one for undergraduate courses and one for graduate courses) and two of them are for course descriptions (one for undergraduate courses and one for graduate courses).

The files should be organized so that for each semester there’s a folder inside the shared courses folder. The folder name should follow the format F1992 for both graduate and undergraduate courses from Fall 1992.

Schedule information

The course schedule for the semester should be kept in a csv file. These files can be opened and edited with Microsoft Excel,1 but also (though a bit more cumbersomely) with any plain text editor. You can use the file sample-sched.csv to get started.

As you will see, each csv file will contain the following information:

  • Course number (without the ‘Phil’ part—e.g. 542, 795F, 100.01, etc.)
  • Course title (best to use full words—i.e. avoid things like ‘Intro to Ethics’ and opt instead for ‘Introduction to Ethics’).
  • Instructor last name.
  • Meeting times (for the sake of consistency, please refer to the list of abbreviations below).
  • Classroom information.

Please make sure not to include any header information on the csv file. As you will see in the example file, each and every row corresponds to information about one course. (Thus there is no row on the file for things like ‘Course No’ or ‘Instructor name’. That information will be generated automatically before we update the Drupal page).

The file should be named using the following convention: F2016-grad-sched.csv for the schedule information for graduate courses from Fall 2016.

Course descriptions

The course descriptions for the semester should be kept in a Markdown file (the extension will be ‘.md’). These files can be opened and edited with any plain text editor. I recommend using the simple in browser editor StackEdit. You can set it up so you have access to the Dropbox folder I have shared with you from within the browser. Other options (if you prefer a standalone app) include Texts and Typora.

The main things to know about editing these files is this:2

  • To specify that a particular word should be italicized, you should surround it with a * or an underscore—e.g. ‘*word*’ (and also ‘_word_’) will be rendered as ‘word’.
  • To specify that a particular word should be in bold, you should surround it with two *s—e.g. ‘**word**’ will be rendered as ‘word’.
  • Em-dashes should be written as ‘---’. En-dashes should be written as ‘--’.

  • For bulleted lists, you simply write a series of lines with a single dash before each of them, as in:

    - First item.
- Second item.
- Third item.

    (For numbered lists, you replace the dashes with the corresponding number)

For each course, the file should contain the following information:

  • Course number (as above: without ‘Phil’.)
  • Course title.
  • Course description.
  • Text information (if provided).
  • Prerequisites (if any).
  • Link to course website (if provided).

For undergraduate courses only, you should also include the following information:

  • GenEd requirements that the course satisfies (if any).
  • Distribution requirements for the major that the course satisfies (if any).

In between any two blocks, please make sure to include a separator. This is a line containing only one of ‘--’ or ‘***’.

The format of each block corresponding to an undergraduate course should be as follows (the three '#'s before the course number, as well as the space after them, are all important):

### 123 | Philosophy course title

Lorem ipsum dolor sit amet, consectetur adipisicing elit, sed do eiusmod
tempor incididunt ut labore et dolore magna aliqua. Ut enim ad minim veniam,
quis nostrud exercitation ullamco laboris nisi ut aliquip ex ea commodo
consequat. Duis aute irure dolor in reprehenderit in voluptate velit esse
cillum dolore eu fugiat nulla pariatur. Excepteur sint occaecat cupidatat non
proident, sunt in culpa qui officia deserunt mollit anim id est laborum.

_Prerequisites_: Two philosophy courses.
_Text_: _The Philosophy Book_

**GenEd**: AL | **Major**: M&E

To avoid confusion with 500-level courses, please specify that prerequisites for those courses apply only to undergraduates. This is somewhat redundant, since course descriptions that include prerequisites will only be listed on the page for undergraduate courses. But in this case, the redundancy will not hurt.

Please make sure to use the same format for course numbers in the schedule file and in the course description file. (For example, use something like ‘100.01’ throughout, and not ‘100.01’ in some places and ‘100-01’ in others.)

Also remember that some courses will be listed in both the graduate and undergraduate pages. Information about prerequisites should only appear on the page for undergraduate courses.

The file should be named using the following convention: F2016-grad-desc.md for the course descriptions for graduate courses from Fall 2016.

Uploading course information

Once you are done editing the csv and md files, you should go to this page. Specify the semester name, whether you are uploading files for undergraduate or graduate courses, and then drag and drop each of the files to the corresponding location.

Click on and copy the resulting html code that will appear on the bottom. (The html code should automatically be copied to your clipboard once it appears, but this may not work depending on which browser you are using.)

Once you have the bit of html copied onto your clipboard, you can go to https://www.umass.edu/philosophy/user and login using your NetID. Navigate to the page you will edit (see below for more details) and click the button at the top right corner.

You should then make sure to click on the button below the ‘Body’ section, delete any text in the resulting box, and paste the contents of your clipboard.

Current semester

There are two pages for the current semester’s courses: one for undergraduate courses and one for graduate courses. Note that these two URLs:

  • https://www.umass.edu/philosophy/undergraduate-courses
  • https://www.umass.edu/philosophy/graduate-courses-philosophy

are currently set to be redirected to their respective ‘current courses’ page, so you should make sure to only make changes to the contents of the two pages at:

  • https://www.umass.edu/philosophy/current-undergraduate-courses
  • https://www.umass.edu/philosophy/current-graduate-courses

You may not need to run the csv and md files again at the beginning of the semester, since all the information may already be available in the ‘future courses’ pages. If so, you can just copy the raw html from that page onto the page for current courses. (Remember: in order to view the raw html of the page you should click on the button.)

Once you change the contents of the ‘Current courses’ page to include the courses for a new semester, you should make sure to ‘unpublish’ the future courses page (see below).

(For example, before the beginning of the Fall 2017 semester, the course information on the ‘Future courses’ pages should be migrated to the ‘Current courses’ page. Since we will not have any information on courses for Spring 2018 for a while, there should not be a ‘Future courses’ page available. So, the easiest thing to do is to ‘unpublish’ the ‘Future courses’ page until we have the course information for Spring 2018.)

Future semester

At some point before registration for the following semester, we should create a page including future courses. In order to create them, please go to one of the following two pages:

  • Future undegraduate courses
  • Future graduate courses

After updating that page, you should make sure that they are published, by scrolling to the bottom of the editing page, selecting the Publishing options tab of the last block on that page, and making sure that ‘Published’ is selected:

You will need to ‘unpublish’ this page once the information it contains is migrated to the corresponding ‘Current courses’ page.

Past semesters

For the time being, we are only keeping one semester’s worth of past courses ‘visible’. I’m still working on a solution for creating a way of allowing the user to navigate to one of many pages, one for each past semester.

Once you replace the contents of the ‘Current courses’ page with those of a new semester, you should put the current courses on one of the two ‘Past courses’ pages, which live here:

  • https://www.umass.edu/philosophy/past-undergraduate-courses
  • https://www.umass.edu/philosophy/past-graduate-courses

Before doing that, you should create a page for archiving the last semester’s course information. To do so, you should create a new page by clicking on ‘Add content’ at the top left corner of any page.

You will then see a list of options for the type of content you are adding. You should create a ‘Basic page’ and paste the raw html in the appropriate place. (Before doing that, remember to click the button.)

You should title the page “Past undergraduate courses” or “Past graduate courses”. You should then scroll down to the bottom of the page, click on the ‘URL path settings’ tab, and make sure the ‘Generate automatic URL alias’ is unchecked.

You should then enter, in the box below ‘URL alias’, the address we will use for the page. Please use the following format: past-[undergraduate or graduate]-courses-[SYYYY]. For example, for creating the archive page for undergraduate courses from Fall 2016, you should write the following as the URL alias: past-undergraduate-courses-f2016.

Example workflow

A few weeks or so before the beginning of Fall 2017, you should do each of the following.

  1. First, create two new pages for archiving the course information from Fall 2016, following the instructions above. Those pages should be named “Past undergraduate courses” and “Past graduate courses”, and they should be given the correct URL alias. You can either run the corresponding csv and md files to create the raw html for each page or you can copy the raw html from the ‘Current courses’ page.
  2. Once the information is archived, you should replace the contents of the ‘Past courses’ pages with that from the corresponding ‘Current courses’ pages. Alternatively, you can either run the corresponding csv and md files to create the raw html for each page. Thus, for example, the body of the page for current undergraduate courses (which will contain information on undergraduate courses for Fall 2016) should be copied and pasted onto the body of the page for past undergraduate courses.
  3. Once you’ve edited the ‘Past courses’ page, you should copy the contents of the ‘Future courses’ page onto the ‘Current courses’ page. Again, you can always run the csv and md files through the scripts to generate the raw html again or you can copy the raw html from the ‘Future courses’ page.
  4. Once you’ve updated the ‘Current courses’ page, you should make sure to ‘unpublish’ the ‘Future courses’ page.
